# Проверка размеров текстур

## Важно! Перед заменой всегда проверяйте размеры

### Текущие размеры SVG:
- **Солнце (sun.svg)**: 216x216 пикселей
- **Луна (moon.svg)**: 127x127 пикселей

### Как проверить размеры оригинальных PNG:

1. Откройте PNG файл в любом редакторе изображений
2. Или используйте PowerShell команду:
   ```powershell
   Add-Type -AssemblyName System.Drawing
   $img = [System.Drawing.Image]::FromFile("путь\к\файлу.png")
   Write-Host "Размер: $($img.Width)x$($img.Height)"
   $img.Dispose()
   ```

### Если размеры не совпадают:

#### Вариант 1: Изменить размер SVG
Отредактируйте SVG файл, изменив атрибуты `width` и `height` в корневом элементе `<svg>`:
```xml
<svg width="ОРИГИНАЛЬНАЯ_ШИРИНА" height="ОРИГИНАЛЬНАЯ_ВЫСОТА" viewBox="0 0 ОРИГИНАЛЬНАЯ_ШИРИНА ОРИГИНАЛЬНАЯ_ВЫСОТА">
```

#### Вариант 2: Настроить масштаб в Godot
1. Откройте SVG файл в Godot
2. В Inspector найдите параметр `svg/scale`
3. Измените значение (например, если PNG был 100x100, а SVG 200x200, установите scale=0.5)

#### Вариант 3: Добавить scale в сцене
Если размеры немного отличаются, можно добавить масштабирование в Sprite2D:
```gdscript
scale = Vector2(МАСШТАБ_X, МАСШТАБ_Y)
```

### Текущая ситуация с солнцем и луной:

✅ **Солнце и луна используются как Sprite2D без явного масштабирования**
✅ **SVG файлы уже импортированы в Godot**
⚠️ **Нужно проверить в игре, совпадают ли размеры**

### Рекомендации:

1. **Запустите игру** и визуально сравните размеры
2. Если размеры отличаются - используйте один из вариантов выше
3. **Сохраните оригинальные PNG** до полной проверки
4. При замене других текстур - всегда проверяйте размеры сначала
